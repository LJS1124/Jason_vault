该技术用于如AUTO DL平台只开放22端口的远程服务器用的，一般是算力平台。
这时候需要本地映射通过22端口来获取请求
##### SSH隧道🌟🌟🌟

- SSH隧道定义：通过SSH协议将本地请求转发到远程服务器的技术
- SSH默认端口：22号端口（但实际端口可能因配置而异）
- 隧道建立条件：需要SSH服务端和客户端建立通信连接
- 隧道工作原理：将其他服务端口（如8080）的请求封装在SSH协议中传输
- 隧道应用场景：当只能访问SSH端口时，通过隧道访问其他网络服务
- 隧道本质：在受限网络环境中建立的安全通信通道

#### 使用案例
 ssh -CNg -L 6006:127.0.0.1:6006 root@123.125.240.150 -p 42151
 解读：
 ### 命令参数解析

将命令拆分后的各参数作用：

- `-C`：启用数据压缩，减少传输流量，提升转发效率。
- `-N`：仅建立 SSH 连接，不执行远程命令（适用于隧道场景，降低资源占用）。
- `-g`：允许本地转发端口被其他主机访问（默认仅本地可连接）。
- `-L 6006:127.0.0.1:6006`：本地端口转发规则，将**本地 6006 端口**的流量，转发到 SSH 服务器（123.125.240.150）上的`127.0.0.1:6006`端口。
- `root@123.125.240.150`：SSH 连接的用户名（`root`）与目标 SSH 服务器地址（`123.125.240.150`）。
- `-p 42151`：指定 SSH 服务器的监听端口为 42151（默认 SSH 端口为 22，此处使用非默认端口）。

### 2. 命令功能

该命令的核心作用是：通过 SSH 加密隧道，将本地机器的 6006 端口与目标 SSH 服务器的`127.0.0.1:6006`端口建立映射。

当本地访问`127.0.0.1:6006`（或其他主机访问该机器的 6006 端口）时，流量会通过 SSH 加密传输到目标服务器，并转发至其本地的 6006 端口服务。
